# Defending ML-IDS Against Backdoor Attacks

Modern Network Intrusion Detection Systems (IDS) have evolved to use deep learning techniques to effectively detect malicious intruders[cite: 1]. However, these systems are vulnerable to backdoor attacks where adversaries embed specific trigger patterns into the training data, manipulating the IDS into granting covert bypasses during testing[cite: 1]. In this project, we develop a novel machine unlearning technique to remove these backdoor samples and secure the poisoned ML-IDS[cite: 1].

## Phase 1: Threat Modeling & Detection

Backdoor attacks in network IDSs require triggers that closely resemble the structure of legitimate network packets, such as subtly altering Time-To-Live (TTL) feature values to evade detection[cite: 1]. 

### Mitigation Strategies Implemented:

* **Rule-Based Dataset Cleansing:** Deployed Suricata, a static rule-based IDS, to analyze the poisoned PCAP files, cross-verify mislabeled entries, and correctly flag abnormal SSH brute-force behaviors without generating excessive false positives[cite: 1].
* **Targeted Machine Unlearning:** Used the corrected, clean dataset generated by Suricata to fine-tune the poisoned Multilayer Perceptron (MLP) model, essentially forcing the system to "forget" the backdoor influence without requiring a full model retraining[cite: 1].
* **Complete Threat Rectification:** The un-poisoned ML-IDS successfully achieved a 100% accuracy rate in classifying the backdoor-triggered flows as 'attack', fully rectifying the compromised model while maintaining over 99.2% accuracy on legitimate, benign traffic[cite: 1].
